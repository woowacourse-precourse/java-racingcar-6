# 프리코스 2주차 - 레이싱 카 게임

### 1. Controller

이번 미션에도 MVC 패턴을 따랐습니다.
개인적으로 MVC 패턴이 전체적인 역할을 분리하는데 쉽기 때문에
코드의 가독성을 크게 올려주는 좋은 설계 방식이라고 생각합니다.
View와 Model을 철저하게 분리하고 해당 로직들을 Controller에서
조절할 수 있도록 하였습니다.


### 2. Model - 도메인 설계

이번 미션에서는 1주차와 다르게 좀 더 객체지향적으로 구성하기 위해서
각 클래스에 대한 역할을 좀 더 꼼꼼히 나누었습니다.

(1) Car
- DTO 역할을 담당하는 클래스입니다.
- 레이싱 게임을 할때 필요한 정보인 차의 이름과 이동한 거리에 대한 데이터를 가지고 있습니다.

(2) GameSetter
- 게임 시작 전, 플레이어로부터 차의 이름과 시도할 횟수를 입력 받는 역할을 합니다.
- 위의 두 가지 데이터를 입력을 받으면 게임을 실행할 준비가 끝납니다.

(3) RacingGame
- 게임을 진행하는 클래스입니다.
- RandomNumberGenerator 객체를 포함하고 있어서, 규칙에 따라 각 차가 이동할지 이동하지 않을지를 결정합니다.

(4) RandomNumberGenerator
- 게임 진행을 도와주는 클래스입니다.
- 게임 진행시 필요한 난수를 생성해주는 클래스입니다.
- 앞서 언급하였듯이 RacingGame 클래스에 속해 있습니다.

(5) WinnerChecker
- 게임의 마무리를 하는 클래스입니다.
- 각 차의 이동 거리를 확인하여 최종 우승자를 뽑아줍니다.

### 3. View
여기서도 View는 Console창에 해당 내용을 print하는 것입니다.
Message 클래스는 싱글톤 패턴으로 구현하여 게임을 재시작할 때마다 새로운 객체가 생성되는 도메인 클래스들과 달리,
한번 인스턴스가 생성되면 계속해서 생성이 될 수 있도록 하였습니다.
