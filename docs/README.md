## 구현해야 하는 기능 목록

### 게임 진행

게임은 아래와 같은 4단계를 거쳐서 실행된다.

```자동차 생성 -> 시도 횟수 입력 -> 경주 진행 -> 결과 확인```

#### 자동차 생성

사용자에게 자동차 이름을 입력받아 자동차를 생성하는 단계이다.

- 제약 조건
    - 1대 이상의 자동차가 존재해야 한다.
    - 자동차 이름에 공백이 들어오면 안된다.
    - 자동차 이름은 1~5자 사이여야 한다.

#### 시도 횟수 입력

사용자에게 몇 번의 이동을 할 것인지 입력받는 단계이다.

- 제약 조건
    - 시도 횟수는 숫자여야 한다.
    - 입력이 1 이상의 수로 들어와야 한다.

#### 경주 진행

각 자동차별로 전진 여부를 결정하여 전진 시키고 각 시도마다 현재 위치를 보여주는 단계이다.

- 제약 조건
    - 전진 여부는 0~9 사이의 랜덤 값을 생성하여 결정한다.
        - 값이 4 이상이면 전진한다.
        - 값이 3 이하이면 멈춘다.

#### 결과 확인

가장 많이 전진한 자동차의 이름을 출력하는 단계이다.

- 제약 조건
    - 시도 횟수는 숫자여야 한다.
    - 입력이 1 이상의 수로 들어와야 한다.
    - 우승자는 한명 이상일 수 있다.
    - 공동 우승자가 존재할 경우 쉼표(,)로 구분한다.

## 체크리스트

### 기능 구현

#### 자동차 생성

- [x]  게임 오프닝 출력 기능 - "경주할 자동차 이름을 입력하세요.(이름은 쉼표(,) 기준으로 구분)" 멘트를 출력한다.
- [x]  자동차 이름 입력 기능 - 쉼표(,)로 구분하여 경주할 자동차 이름을 입력받는다.

#### 시도 횟수 입력

- [x] 시도 횟수 입력 요청 멘트 출력 기능 - "시도할 회수는 몇회인가요?" 멘트를 출력한다.
- [x] 시도 횟수 입력 기능 - 몇번의 이동을 진행할 것인지 입력받는다.

#### 경주 진행

- [x] 경주 진행 기능 - 각 자동차별로 0~9 사이의 난수를 생성하여 4이상이면 한칸 전진하여 총 이동거리를 기록한다.

#### 결과 확인

- [x] 실행 결과 출력 안내 기능 - "실행 결과" 멘트를 출력한다.
- [x] 실행 결과 출력 기능 - 각 자동차별 이동 거리를 출력한다.
- [x] 최종 우승자 출력 기능 - 최종 우승자를 출력한다. (단, 공동 우승자가 존재할 경우 쉼표로 구분하여 모두 출력한다.)

### 추가 요구 사항

- [ ] 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- [ ] JUnit 5와 AssertJ를 이용하여 본인이 정리한 기능 목록이 정상 동작함을 테스트 코드로 확인한다.
