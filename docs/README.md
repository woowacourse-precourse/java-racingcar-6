## 🔍 숫자 야구 게임 요구사항

### 📝1. 자동차 경주 게임 구현

```sh
1. 사용자로부터 자동차 이름(쉼표로 구분)을 입력 받아야 합니다.
2. 사용자로부터 시도할 회수를 입력 받아야 합니다.
3. 각 차수별로 자동차들은 전진 또는 멈추는데, 전진 조건은 0에서 9 사이의 무작위 값이 4 이상일 때 입니다.
``````
<br>

### 📋2. 입력 및 출력 요구 사항

```sh
1. 자동차 이름은 5자 이하로 제한되며, 쉼표(,)로 구분합니다.
2. 각 차수별 실행 결과를 출력해야 합니다.
3. 경주가 종료된 후, 우승자(또는 공동 우승자)를 알려줘야 합니다.
4. 잘못된 입력 값이 들어오면 IllegalArgumentException을 발생시키고 애플리케이션을 종료해야 합니다.
``````
<br>

### ⚡3. 프로그래밍 요구 사항

```sh
1. JDK 17에서 실행 가능해야 합니다.
2. main() 메서드를 시작점으로 사용해야 합니다.
3. build.gradle 파일을 변경할 수 없습니다.
4. 외부 라이브러리를 사용할 수 없습니다.
5 .Java 코드 컨벤션을 준수해야 합니다.
6. System.exit()를 호출하지 말아야 합니다.
7. ApplicationTest의 모든 테스트가 성공해야 합니다.
8. indent depth를 3 이하로 유지해야 합니다.
9. 3항 연산자를 사용하지 말아야 합니다.
10. 함수(또는 메서드)가 한 가지 일만 하도록 작게 만들어야 합니다.
11. JUnit 5와 AssertJ를 사용하여 테스트 코드 작성해야 합니다.
``````

<br>

### ⚡4. 구현한 클래스와 기능

```sh
1. Car 클래스
=>
Car 클래스는 자동차의 속성과 행동을 정의합니다. 자동차의 이름, 위치, 그리고 경주 보고서를 관리합니다.
move() 메서드는 무작위로 생성된 숫자에 따라 자동차를 움직이는 역할을 합니다.
statusReport() 메서드는 자동차의 현재 위치와 보고서를 출력합니다.

2. CarStart 클래스
=>
CarStart 클래스는 경주 게임을 시작하고 진행하는 기능을 제공합니다.
lineUp() 메서드는 사용자로부터 자동차 이름을 입력받고, 유효성 검사를 수행한 후 자동차 객체를 생성하여 리스트에 저장합니다.
inputRound() 메서드는 사용자로부터 경주 라운드 수를 입력받고, 유효성 검사를 수행한 후 라운드 수를 설정합니다.
raceStart() 메서드는 입력받은 라운드 수에 따라 자동차를 움직이고, 보고서를 출력합니다.
awards() 메서드는 경주에서 우승한 자동차를 결정하고, 우승자를 출력합니다.
findMaxPosition() 메서드는 경주 중 가장 멀리 이동한 자동차의 위치를 찾습니다.

3. Validator 클래스
=>
Validator 클래스는 사용자 입력 데이터의 유효성을 검사하는 메서드를 제공합니다. 
이 클래스는 중복된 이름, 빈 입력값, 이름 길이, 숫자 여부 등을 검사합니다.

4. Application 클래스:
=>
Application 클래스는 프로그램을 실행하는 엔트리 포인트를 정의합니다. 
CarStart 객체를 생성하고 경주 게임을 시작합니다.
``````
