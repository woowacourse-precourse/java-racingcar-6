## 📝 요구 사항 정리

### 컴퓨터 (경주 진행자)

**자동차**

- 자동차 관리 객체
- 각 자동차 별 전진 여부에 따라 카운트 증가
- 자동차 이름과 카운트를 출력으로 전달
- 카운트에 따라 우승자 판단
  - 여러명인지에 대해서도 판단

**레이스**

- random 값 관리 객체
- 0에서 9 사이의 random 값 생성
- random 값이 3 이하면 false(멈춤), random 값이 4 이상이면 true(전진) 반환

### 플레이어

- 자동차 이름을 부여한다.
  - 이름이 두개 이상인지
  - 쉼표(,)를 기준으로 구분
  - 이름은 5자 이하만 가능
  - 공백 금지
  - 중복 금지
- 주어진 회수 입력
  - 숫자인지 아닌지
  - 1 이상의 값인지 (0이면 경주 진행 안됨)

## 🔗 도메인 설계

**Controller 패키지**

- **RaceController**: 자동차 경주에 대한 컨트롤러

**Domain 패키지**

- **Car**: 자동차의 정보에 대해 관리하는 클래스

  - 필드
     - 자동차 이름
     - 자동차 위치
  - 메서드
      - 자동차 이동시키기
          - 4 이상의 값을 인자로 받으면 자동차의 위치를 1 증가시킨다.

- **RacingCars**: 생성한 Car 리스트를 관리하는 일급 컬렉션

  - 필드
      - Car 리스트
  - 메서드
      - 자동차 이름에 대한 유효성 검사
          - ,를 기준으로 자동차 이름을 split 하여 리스트로 반환
      - 자동차 리스트에 있는 모든 자동차를 경주시킨다.

- **Race**: 경주에 대해 관리하는 클래스.

  - 필드
     - 사용자의 시도 횟수 (경주 횟수)
  - 메서드
     - 경주가 시도 횟수만큼 진행되었는지 확인한다.

- **RaceNumberGenerator**: 레이스를 위해 필요한 0~9 사이의 값을 랜덤하게 뽑는 클래스

- **WinnerCar**: 우승한 자동차에 대해 관리하는 클래스.

  - 메서드
      - 가장 먼 위치의 자동차를 찾아 자동차의 이름을 담은 리스트를 리턴한다.
      - 가장 큰 position(위치)를 찾는다

**Massage 패키지**

- **ExceptionMessage**: 예외 메시지
- **InputMessage**: 입력값을 받기 위해 안내하는 메시지
- **OutputMessage**: 출력에 필요한 메시지

**Exception 패키지**

- 자동차 예외 클래스
- 회수 예외 클래스

**Service 패키지**

- **CarsService**: Cars의 정보를 처리하여 컨트롤러에게 전달하기 위한 중간 서비스 클래스.
- **RaceService**: Race의 정보를 처리하여 컨트롤러에게 전달하기 위한 중간 서비스 클래스.
- **WinnerCarService** : WinnerCar의 정보를 컨트롤러에게 전달하기 위한 중간 서비스 클래스.

**View 패키지**

- **InputView**: 콘솔을 통해 사용자에게 값을 입력받고, 에러 발생 시 예외 발생시키고 종료하는 클래스.
- **OutputView**: 콘솔에 대한 출력을 담당하는 클래스.
